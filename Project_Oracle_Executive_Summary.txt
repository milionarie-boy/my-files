1. THE STRATEGIC PROBLEM
================================================================================
"Charts are becoming a commodity."

The retail and institutional trading landscapes are shifting toward massive
automation. TradingView dominates charting but lacks cognitive intelligence
- the ability to generate strategies from natural language, detect patterns
visually, and provide real-time sentiment awareness.

Current gap: Users must learn Pine Script manually, identify patterns with
their eyes, and switch between tools for sentiment data.

Opportunity: Dante Labs can move from descriptive charting to systematic
cognitive intelligence - creating a proprietary, high-margin AI tier that
increases stickiness and subscription value.


2. THE SOLUTION: PROJECT ORACLE
================================================================================
A multimodal AI layer embedded directly into TradingView's visual engine,
consisting of two core AI models working in parallel:

COMPONENT 1: LANGUAGE (PineCoder-70B)
- Model: Llama-3-70B (4-bit quantized)
- Training: 120,000 Pine Script v5 examples + official documentation
- Fine-tuning: LoRA on NVIDIA H100 clusters
- Function: Converts plain English into executable Pine Script v5 code

COMPONENT 2: VISION (ChartTopology-V1)
- Model: CNN + Vision Transformer
- Function: Interprets chart pixels to detect patterns
- Patterns detected: Head & Shoulders, Wyckoff, Elliott Waves, S/R zones
- Output: JSON coordinates + confidence score (0.0 to 1.0)

USER OUTCOMES:
- Describe entry rules in English -> AI generates backtested strategy instantly
- Chart automatically labeled with pattern boundaries
- Real-time sentiment index (Bearish/Bullish) from X, Discord, and News
- One-click deploy to broker connection

4. HOW IT WORKS: ARCHITECTURE
================================================================================
[Binance/CME] --WebSocket--> [Ingestion Gate]
                                  |
                                  v
[User Chat/View] ------gRPC----> [DANTE ORACLE CORE]
                                  |              |
                                  v              v
                          [LLM Inference]  [Vision AI]
                           (PineCoder-70B) (ChartTopology)
                                  |              |
                                  v              v
                          [Signal Aggregator] <--+
                                  |
                                  v
                  [TradingView Chart UI] --UDF/Canvas-->

STACK:
- Language: Golang (streaming) + Python (inference)
- Message Broker: Apache Kafka
- Database: ClickHouse (time-series for backtesting)
- Orchestration: Kubernetes on AWS/GCP with GPU auto-scaling
- Inference Server: NVIDIA Triton with TensorRT optimization

INTEGRATION WITH TRADINGVIEW:
- Universal Data Feed (UDF) for real-time signals
- Canvas API for AI overlays (confidence bands, price paths)
- Webhooks for one-click broker deployment
- React-based side panel embedded via TradingView's widget API
